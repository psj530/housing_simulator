# 프로젝트 이관 요약 — “예비신혼부부 시뮬레이션 허브”

> 새 Claude 계정의 첫 대화(또는 Claude 프로젝트의 Files)에 이 문서를 붙여넣고, 함께 `index.html`을 업로드하면 이어서 작업할 수 있어요. 응답은 한국어, 따뜻하고 간결하게.

---

## 1) 프로젝트 개요
예비신혼부부(신랑 **홍지훈** / 신부 **박수정**)를 위한 **단일 HTML 웹앱**. 결혼 준비부터 내 집 마련(대출)까지 한 화면에서 슬라이더·직접입력으로 계산하는 **참고용 도구**. 백엔드 없이 브라우저에서만 계산(개인정보 서버 전송 없음). 모든 금액 단위 = **천원**.

## 2) 대상 부부 정보·조건
- 부부합산 연소득 **1.3억**(지훈·수정 각 6,500만), 자기자본 **4.7~5억**(저축 3.2억 + 증여 1.5억).
- 서울·**생애최초**·실거주, 1년 이내 매수 목표. 국민평수(전용 84) 포함, 예산이 핵심.
- 양가: 처가 **인천 미추홀** / 시가 **독산한신(금천 독산)**. 직장: **대방(동작)**·**용산(한강로동)**.
- 요구: 실거주+점프업 기대, 어느 정도 인프라, 차량 1대+ 주차. (1호선 조건은 제거됨)
- 증여 한도(참고): 1인 = 기본 5,000만 + 혼인공제 1억 = 1.5억, 부부 합산 최대 3억 비과세.

## 5) 탭 구조 (6개, SPA)
순서: **내 집 마련(첫 화면·기본 활성)** → 결혼 준비 → 하객 → 신혼여행 → 상견례 → 종합.
- **결혼 준비**: 항목 슬라이더(예식장·스드메·예물·혼수·신혼여행·기타) 합산 + 예비비 10% + 추천 예식장 5곳(그랜드컨벤션 당산·스타시티아트홀 건대·빌라드지디 수서·라마다 신도림·더컨벤션 반포).
- **하객**: 참석수·1인식대·축의금·납부비율 → 순수지(축의금−식대).
- **신혼여행**: 목적지 프리셋 + 박수·항공·숙박·현지·기타 → 총경비/1인당.
- **상견례**: 팁 카드 4종 + 식사비(인원×단가) 계산.
- **내 집 마련**: 아래 6~9절.
- **종합**: 각 탭 값 실시간 집계(R 레지스트리 + `refreshSummary()`). 결혼 순지출 vs 내 집 마련, ‘결혼비용을 자기자본에서 쓰면’ 연결 인사이트.

## 6) 내 집 마련 — 4분할 레이아웃(2×2)
`.grid2`(2열) 안에 `.lstack`(왼쪽) / `.rstack`(오른쪽) flex 컬럼:
- **좌상 우리 자금**: 지훈·수정 각각 연소득·저축·증여·차용증·퇴직금 슬라이더(2열, `data-loan=income_g/income_b/save_g/…`) + ‘퇴직금 합산 포함 ☐지훈 ☐수정’(`sevchk_g/b`) + **합산 자본금 박스**(`#capsum`: 합산 연소득 / 저축·증여·차용 / 총 동원 자금).
- **우상 결과**: 주담대·주택가격·취득세·기타부대비·자기자본·필요대출·LTV한도·규제한도(6억cap)·DSR/현재DSR·부족분.
- **좌하 주택·대출 조건**: 주택가격·금리·**만기(10~30, 30 상한)**·신용대출·상환방식(원리금/원금/체증)·생애최초(70/40)·금리유형(변동/고정혼합).
- **우하 상환 분석**: 상환방식별 월납입/총이자(`#l-methods`) + 이율 ±1%p 민감도(`#l-sens`).
- sticky는 제거(4분할 정적 격자). 모바일 640px↓ 1열.

## 7) 대출 계산 로직 (검증 완료)
`computeLoan(income, equity, price, rate, term, credit, sev, first, fix)`:
- 취득세율: ≤6억 1.1% / 6~9억 2.2% / >9억 3.3%(근사). 기타부대비 0.7%.
- LTV: 생초 70% / 일반 40%. hardCap: ≤15억→6억 / ≤25억→4억 / else 2억.
- **stressRate = rate + (변동 3.0 / 고정·혼합 2.4)** ← 2026 하반기 3단계, 혼합형 80%.
- DSR 40%, 만기 30년 상한. `equity` = 저축+증여+차용, `sev` = 포함 퇴직금.
- 상환방식: `equalPay`/`principalPay`/`gradPay`(체증 +3%/년).
- 검증 예: 주택 10.7억·자기자본 4.7억 → 필요 6.43억, 6억cap, 부족 ~4,280만, DSR 38.7%.

## 8) 2026 정책 반영 (서울·수도권·규제지역 기준)
6.27 대책: 수도권·규제지역 주담대 **6억 한도**, 생초 LTV 70%(전입의무), 만기 30년. 10.15: 규제지역 확대(강남·서초·송파·용산 등). **2026 하반기(7/1~12/31) 스트레스 DSR 운영방안**: 수도권·규제지역 **3단계(하한 3.0%p)**, 지방 2단계 유지. 안내문·정책 기준 주석에 반영. (툴은 ‘서울=규제지역·무주택 40%’로 단순화 — 비규제 수도권은 50%, 규제/비규제 토글은 향후 과제.)

- **차량 소요(추정)**: 4곳(처가 미추홀 37.4639,126.6503 / 시가 독산 37.4718,126.8975 / 대방 37.5134,126.9265 / 용산 37.5299,126.9648)까지 haversine×1.3, 구간 평속 24/30/40km/h. “직선거리 기반 추정” 명시.
- ‘시뮬레이터에 넣기’/‘넣기’ → 주택가격 슬라이더 반영.

## 10) 지도 (① · ② 비교)
OSM `export/embed.html`(키 없음). 두 슬롯 `#map1`/`#map2` 비교 + 비교 패널(매매가·전용·흐름·차량소요). 검색창(`#map-q`)→Nominatim 지오코딩→① 슬롯. 정밀 위치는 각 매물 ‘네이버지도 ↗’. **구글 `output=embed`는 차단됨**, 네이버는 키 필요라 미채택.

## 16) 작업 방식·제약 (선호)
- 응답 한국어, 따뜻·간결. 모든 시뮬레이션에 면책(참고용 추정, 금융·세무 자문 아님, 진행 전 은행·세무사·중개사 확인).
